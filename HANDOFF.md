## Bugs fixed in the final stretch

1. **`options_code_agent` had unescaped `{ }` in its prompt template** → `.format()` crashed on `KeyError`. Fixed by doubling braces (`{{` / `}}`).
2. **`tcn_intraday_seq` generated code used deprecated `fillna(method=...)`** → validator caught it correctly. Added explicit warning to all code-agent prompts.

If you see `KeyError` on a prompt format, look for unescaped `{ }`.
If you see `fillna() got an unexpected keyword argument 'method'`, that's a
Claude-generated bug — re-run, Claude will likely get it right next time.

## Architecture quick reference

```
research layer       validation layer       execution layer
─────────────────    ──────────────────    ────────────────────
research_agent       backtesting_agent      code_agent
autonomous_agent     risk_agent             ml_code_agent
ml_research_agent                           options_code_agent
options_research_agent                      execution_agent
                                            options_agent
                                            monitor_agent
```

**12 distinct agents**, each with a registry-aware prompt (won't propose
duplicates of strategies already in `STRATEGIES`).

### The 3 strategy types — each module exports

| Type        | Generated by         | Contract                                                          |
|-------------|----------------------|-------------------------------------------------------------------|
| Rule-based  | `code_agent`         | `signals(df, params) -> pd.Series` (pure indicator math)          |
| ML          | `ml_code_agent`      | `signals(df, params) -> pd.Series` (trains inside on first half) |
| Options     | `options_code_agent` | `signals(df, params)` + `options_intent(sig, px, params)`         |

All three feed the same backtest engine → comparable Sharpe / $ PnL / WR.

## CLI flags reference

| Flag                    | Effect                                                                   |
|-------------------------|--------------------------------------------------------------------------|
| (none)                  | Reads cached ideas, no SDK calls                                         |
| `--refresh-research`    | Fresh SDK calls on all 4 research agents (research, autonomous, ml, options) |
| `--implement-autonomous`| `code_agent` fires on unmatched autonomous + research ideas              |
| `--implement-ml`        | `ml_code_agent` fires on `ml_research_agent` proposals                   |
| `--implement-options`   | `options_research_agent` + `options_code_agent` fire                     |
| `--tickers all`         | Use every parquet in `DATA_DIR` (~20 tickers)                            |
| `--tickers SPY,QQQ,...` | Custom comma-separated list                                              |
| `--quick`               | 3 tickers + 2-year window                                                |
| `--no-wf`               | Skip walk-forward (saves ~30 min on full run)                            |
| `--strategies a,b,c`    | Test only specific strategy keys (skips research/match phases)           |
